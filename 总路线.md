# 开发者基本功训练总路线

目标不是背命令，而是把常见开发工作拆成能独立完成的小动作：找文件、改文件、查历史、写脚本、调接口、处理数据、连接远程环境，最后组合成一个质量很高的工程化小项目。

## 主题轮换

1. Linux / Bash
2. Git
3. Python 脚本
4. 正则表达式
5. HTTP / API
6. Docker 基础
7. VSCode / SSH
8. 数据处理

每轮都保持 5 个固定部分：

- 大白话解释概念
- 一个 60 分钟以内的小任务
- 可以直接照着敲的命令示例
- 真实开发中的用途
- 一个自测问题

## 递进方向

### 阶段 1：能动手

重点是把电脑里的文件、命令、Git 记录和简单脚本跑起来。这个阶段不用追求优雅，先建立“我能查、能改、能验证”的感觉。

产出示例：

- 用 Bash 找出项目里的大文件、空文件、最近修改文件
- 用 Git 建立清晰提交
- 用 Python 批量整理文本

### 阶段 2：能串联

开始把单点技能连起来：脚本读取文件，正则抽取信息，HTTP 请求拿数据，Git 记录变更。

产出示例：

- 一个命令行小工具：扫描目录、提取日志、输出报告
- 一个 API 调用脚本：请求接口、处理 JSON、保存 CSV

### 阶段 3：能交付

把工具放进可复用环境：Docker 化、README 写清楚、VSCode 远程调试、数据结果可复现。

产出示例：

- 一个可运行的开发者工具项目
- 包含 README、示例数据、测试、Dockerfile、使用说明

### 阶段 4：优秀项目

最终项目建议方向：`dev-workbench`，一个面向个人开发流程的小工具箱。

它可以包含：

- 文件巡检：统计文件数量、大小、最近修改时间
- Git 巡检：展示最近提交、未提交文件、分支状态
- 日志分析：用正则提取错误、接口耗时、状态码
- API 检查：请求健康检查接口并生成报告
- 数据处理：把结果导出为 Markdown / CSV
- Docker 运行：任何机器上都能复现

## 调整机制

你每次放在练习文件夹里的 `.txt` 反馈会作为下一次的依据：

- 如果你觉得概念抽象，下次先加更多类比和反例。
- 如果你觉得任务太简单，下次增加一个真实项目约束。
- 如果你觉得命令记不住，下次减少新命令数量，强化复现。
- 如果你遇到具体报错，下次优先围绕这个报错设计练习。
